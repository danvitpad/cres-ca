# Карта вертикалей CRES-CA

Цель: под каждую индустрию свой набор дефолтов — шаблоны услуг, форма анамнеза (или её отсутствие), тип ценообразования, типовая структура команды, документы клиента, расходники, KPI. Без этой карты мы рискуем навязать бьюти-логику тем, кому она не нужна (грумерам, репетиторам, сантехникам).

Легенда:
- ✅ — да, по умолчанию включено
- ⚠️ — опционально, мастер сам решает
- ❌ — не используется

## 1. Beauty (красота)

**Профессии:** парикмахер, колорист, барбер, мастер ногтей, брови/ресницы, визажист, косметолог, депиляция, массажист (эстетический), тату.

| Аспект | Дефолт |
|---|---|
| Анамнез (аллергии, хрон.) | ✅ обязателен — особенно косметология, татуаж, депиляция |
| Противопоказания | ✅ шаблонный список (беременность, сахарный диабет, онкология, кардио) |
| Фото до/после | ✅ ключевая фича (portfolio, feed, IG-парсинг) |
| Тип услуг | fixed price + длительность; опции (длина волос, сложность) |
| Типовая длит. | 30–180 мин |
| Предоплата | ⚠️ часто для новых клиентов и сложных услуг (30–50%) |
| Расходники | ✅ товарный учёт (краска, лак, воск, иглы) |
| Оборудование | ⚠️ только для лазерной/аппаратной |
| Команда | индивид + салон + аренда кресла |
| Реcook цикл (re-book) | 21–28 дн (волосы, ногти) — smart reminder |
| Документы | сертификаты мастера (для клиента — nothing обычно) |
| Рассылки | ✅ акции, burning slots, "продукт после визита" |
| Чувствит. данные | ✅ мед.анамнез под GDPR special category |

**Шаблоны услуг (defaults при создании аккаунта):**
- Женская стрижка · 60 мин · fixed
- Мужская стрижка · 45 мин · fixed
- Окрашивание (сложное) · 180 мин · fixed+мат.
- Маникюр классика · 60 мин · fixed
- Педикюр · 75 мин · fixed
- Наращивание ресниц · 120 мин · fixed
- Чистка лица · 90 мин · fixed

## 10. Особые / вне сетки

Стоит выделить, т.к. не вписываются в 9 предыдущих:

### Tattoo / Piercing (сейчас в beauty, но специфика отдельная)
- Обязательное информированное согласие (несовершеннолетних нельзя)
- Противопоказания: беременность, гепатит, ВИЧ, эпилепсия
- Фото работ — ключевое
- Предоплата обязательна
- Очень долгие сеансы (4–8 ч), бронь сессиями

### Tutoring online (подраздел education)
- Онлайн-формат → линк на Zoom/Meet
- Платформа не генерирует линк, но хранит и отправляет
- Recurring обязателен

### Marketplace / выездные услуги (cross-industry)
- Home, events, pet-walking, auto-detailing — все "выездные"
- Адрес клиента, расчёт км, возможно минимальный чек
- Нужен флаг `is_on_site_service` в услуге

## Что отсюда вытекает для кода

### 1. Onboarding (выбор вертикали при регистрации мастера)
- Сейчас мастер при регистрации выбирает `specialization` свободным текстом. Нужно **сначала** выбрать вертикаль → потом профессию внутри вертикали.
- После выбора подгружаем `vertical_defaults` (шаблон услуг, формы, включённые фичи).

### 2. Таблица `vertical_profiles`
```sql
CREATE TABLE vertical_profiles (
  key text PRIMARY KEY, -- 'beauty', 'health', ...
  name_i18n jsonb,
  features jsonb, -- { anamnesis: true, before_after: true, equipment_lock: false, ... }
  default_services jsonb -- array of templates
);
```
+ `masters.vertical_key` (FK на vertical_profiles).

### 3. Условная анамнез-форма
`/forms` сейчас универсальная. Нужно:
- Посмотреть `vertical_key` мастера, на бронь к которому пришёл клиент;
- Если у вертикали `features.anamnesis === false` — не вызывать `/forms?prompt=intake` вообще;
- Если `features.anamnesis === 'extended'` (health) — показывать расширенный опросник.

### 4. Специализированные сущности
- `pets` (client_id, name, species, breed, dob, vaccinations, notes)
- `vehicles` (client_id, make, model, year, plate, odometer, color)
- `events` (client_id, event_date, guest_count, venue, style)

Бронь ссылается на **одну из этих сущностей опционально**: `appointment.subject_pet_id | subject_vehicle_id | subject_event_id`.

### 5. Модель услуги
Сейчас услуга — `{ name, duration, price, currency }`. Нужно:
- `pricing_model`: `fixed | per_hour | per_sqm | per_item | per_person | on_request`
- `is_on_site_service` — выезд
- `is_online` — формат онлайн + link placeholder
- `subject_kind`: `client | pet | vehicle | event | group`

### 6. Расходники и equipment
- Beauty/home/auto — `consumables` linkage на услугу (уже есть `products`, надо довязать).
- Health/auto/fitness — `equipment_locks` на слоте (есть миграция, не подключено в booking UI — это было в gap-analysis).

### 7. Feed
- Тип поста `burning_slot` **не показывать** для health-вертикали (реклама мед.услуг регулируется).
- Тип поста `promotion` — для health тоже с осторожностью, скорее "напоминание о чек-апе".

### 8. Подписки и smart re-book
- `recurring cycle days` задаётся per-vertical: beauty=21, wellness=10, pet_grooming=45, dental=180.

### 9. Интерфейс календаря
- Events — по дням, не часам.
- Fitness/Wellness group — по классам с кол-вом мест.
- Education recurring — цикл уроков с авто-генерацией.

**Интеграция (X4):**
1. `/onboarding/create-business` читает `?vertical=X` из query.
2. После создания `masters` / `salons` row показывает preview-экран с галочками — все услуги пре-выбраны.
3. Submit → bulk `supabase.from('services').insert(...)` с `master_id` и `vertical` полем.
4. Если мастер снял все галочки или пропустил — переход к следующему шагу.

Код массива дублируется в `.knowledge/verticals.md` (источник правды) и в `src/lib/verticals/default-services.ts` (runtime). При правке — менять оба файла.
