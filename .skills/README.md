# .skills/ — project-specific skills для CRES-CA

Категории совпадают с `D:/toolbox/skills/` (frontend/backend/research/data/devops/testing).

**Правило:** если skill уже универсальный — держим в `D:/toolbox/skills/<category>/`, сюда кладём только CRES-CA-специфичные (например, привязанные к конкретной таблице / RPC / вертикали). Всё, что становится универсальным, при одобрении уезжает в toolbox по §14 auto-promote.
